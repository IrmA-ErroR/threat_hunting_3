### Задание 2: Надите утечку данных 2

Другой атакующий установил автоматическую задачу в системном
планировщике cron для экспорта содержимого внутренней wiki системы. Эта
система генерирует большое количество трафика в нерабочие часы, больше
чем остальные хосты. Определите IP этой системы. Известно, что ее IP
адрес отличается от нарушителя из предыдущей задачи.

``` r
print(
  data %>%
    select(timestamp, src, dst, bytes) %>%
    mutate(timestamp = hour(as_datetime(timestamp/1000))) %>%
    filter(!grepl('^(12|13|14).*', dst) & timestamp >= 0 & timestamp <= 15 & src != "13.37.84.125") %>%
    group_by(src) %>%
    summarise(bytes_amount = sum(bytes)) %>%
    slice_max(order_by = bytes_amount, n = 1) %>%
    pull(src)
)
```

    [1] "12.55.77.96"

Этот запрос выбирает данные с полями временной метки, источника,
назначения и объема данных. Затем он преобразует временные метки в часы,
фильтрует данные по условиям, группирует их по источнику, суммирует
объемы данных для каждого источника, выбирает источник с наибольшим
объемом данных и выводит его

### Задание 3: Надите утечку данных 3

Еще один нарушитель собирает содержимое электронной почты и отправляет в
Интернет используя порт, который обычно используется для другого типа
трафика. Атакующий пересылает большое количество информации используя
этот порт, которое нехарактерно для других хостов, использующих этот
номер порта. Определите IP этой системы. Известно, что ее IP адрес
отличается от нарушителей из предыдущих задач.

``` r
print(
  data %>%
    select(src, port, dst, bytes) %>%
    filter(!str_detect(dst, '^(12|13|14).')) %>%
    group_by(src, port) %>%
    summarise(bytes_ip_port = sum(bytes), .groups = "drop") %>%
    group_by(port) %>%
    mutate(average_port_traffic = mean(bytes_ip_port)) %>%
    ungroup() %>%
    slice_max(order_by = bytes_ip_port / average_port_traffic, n = 1) %>%
    pull(src)
)
```

    [1] "12.30.96.87"

Запрос анализирует данные сетевого трафика, фильтрует их по условию,
суммирует объемы данных для каждой уникальной комбинации источника и
порта, вычисляет средний объем трафика для каждого порта, а затем
выводит источник с самым высоким отношением объема трафика к среднему
объему трафика для этого порта.

### Задание 4: Обнаружение канала управления

Зачастую в корпоротивных сетях находятся ранее зараженные системы,
компрометация которых осталась незамеченной. Такие системы генерируют
небольшое количество трафика для связи с панелью управления бот-сети, но
с одинаковыми параметрами – в данном случае с одинаковым номером порта.
Какой номер порта используется бот-панелью для управления ботами?

``` r
port_n <- data %>%
  group_by(port) %>%
  summarise(minBytes = min(bytes),
            maxBytes = max(bytes),
            diffBytes = max(bytes) - min(bytes),
            avgBytes = mean(bytes),
            count = n()) %>%
  filter(avgBytes - minBytes < 10 & minBytes != maxBytes) %>%
  pull(port)

print(port_n)
```

    [1] 124

Этот запрос группирует данные по порту, вычисляет минимальное,
максимальное, разницу, среднее значение и количество записей для каждого
порта. Затем он фильтрует порты, у которых разница между средним и
минимальным объемом данных меньше 10 и минимальный объем данных не равен
максимальному, и выводит список таких портов.

### Задание 5: Обнаружение P2P трафика

Иногда компрометация сети проявляется в нехарактерном трафике между
хостами в локальной сети, который свидетельствует о горизонтальном
перемещении (lateral movement).

В нашей сети замечена система, которая ретранслирует по локальной сети
полученные от панели управления бот-сети команды, создав таким образом
внутреннюю пиринговую сеть.

Какой уникальный порт используется этой бот сетью для внутреннего
общения между собой?

``` r
print(
  data %>%
    filter(str_detect(src, "^12\\.|^13\\.|^14\\.") & str_detect(dst, "^12\\.|^13\\.|^14\\.")) %>%
    group_by(port) %>%
    summarise(uniq_port_1 = max(bytes) - min(bytes)) %>%
    arrange(desc(uniq_port_1)) %>%
    slice_head(n = 1) %>%
    pull(port)
)
```

    [1] 115

Этот запрос фильтрует данные по условиям, связанным с источником (src) и
назначением (dst), начинающимися с “12”, “13” или “14”. Затем он
группирует данные по порту, вычисляет разницу между максимальным и
минимальным объемом данных для каждого порта, сортирует результаты по
убыванию этой разницы и выбирает порт с наибольшей разницей

### Задание 6: Чемпион малвари

Нашу сеть только что внесли в списки спам-ферм. Один из хостов сети
получает множество команд от панели C&C, ретранслируя их внутри сети. В
обычных условиях причин для такого активного взаимодействия внутри сети
у данного хоста нет.

Определите IP такого хоста.

``` r
host_ip <- data %>%
  filter(str_detect(src, "^12\\.|^13\\.|^14\\.") & str_detect(dst, "^12\\.|^13\\.|^14\\.")) %>%
  group_by(src) %>%
  summarise(count = n()) %>%
  arrange(desc(count)) %>%
  slice(1) %>%
  pull(src)

print(host_ip)
```

    [1] "13.42.70.40"

Этот запрос фильтрует данные по условиям, связанным с источником (src) и
назначением (dst), начинающимися с “12”, “13” или “14”. Затем он
группирует данные по источнику, подсчитывает количество записей для
каждого источника, сортирует результаты по убыванию этого количества и
выбирает источник с наибольшим количеством записей.
